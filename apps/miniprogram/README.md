# 微信小程序录音端

本目录是原生微信小程序，面向 `COLLECTOR` 采集员，已实现微信快捷登录、数字账号密码登录、独立个人资料与头像、任务权限申请、“待处理 / 已提交 / 已完成”任务数据页、参考源展示、录音、文本补充、审核领取前修改、只读查看和个人统计。小程序身份 ID 固定为 `MINI-...`，角色由服务端鉴权固定为 COLLECTOR，不作为小程序用户文档字段保存。作业页直接读取任务响应中的 `configuration`，不再请求任务版本。`TEXT` 成果允许纯文本、纯录音或两者同时提交；`AUDIO` 必须提交录音且不得夹带文本。

登录页为“砚数声采”A 版，使用正式品牌图标；默认微信快捷登录，可切换 6–12 位非零开头数字账号登录。每个小程序账号只允许一个活动会话；遇到 `ACCOUNT_IN_USE` 时，页面必须先展示确认，再携带返回的短时一次性凭证调用 `POST /api/auth/miniprogram/takeover`，不得自动接管，旧设备后续请求会收到 `SESSION_REPLACED`。微信首次进入不再要求填写姓名；资料不完整仍可浏览任务，但申请权限、查看待办、领取和继续作业会提示并跳转 `pages/profile-settings/index`。头像卡本身是原生 `button open-type="chooseAvatar"`，点击后由微信直接提供头像、相册或拍照等来源，选择完成后先进入应用内预览，确认后才上传。“恢复默认头像”位于同一头像卡底部右侧，确认后恢复本地 `assets/icons/default-collector-avatar.svg` 耳机头像。上传文件仍由现有后端校验为 JPEG/PNG/WebP 且最大 5MB；头像文件不存在时静默回退默认头像，资料成功刷新会清除旧错误提示。

当前账号资料摘要随不透明 token 一起保存在 `recSession.user`。已完善资料时，任务入口直接使用缓存并静默刷新，断网不会误判为资料未完善；没有完整缓存且联网失败时只显示网络 Toast。退出登录和修改密码重新登录会删除整个会话缓存，微信登录、数字账号登录或接管成功会整体覆盖旧账号资料。姓名、资料完成状态及头像元数据更新后同步写回缓存，但不额外缓存头像二进制文件。

“我的”统计页不显示用户 ID，只保留姓名、资料状态和资料设置入口；资料设置页的基本信息区继续展示完整 `MINI-...` userId，直接点击整行会复制完整值并提示“用户 ID 已复制”，剪贴板失败或 ID 缺失时提示“复制失败，请重试”。个人页同时展示最近 5 条提交记录及其状态。

任务大厅不展示全局“当前有未完成作业”卡片，也不缓存全局当前条目。每个任务通过“查看任务”进入自己的“任务数据”页；每次点击“领取新数据”都会使用新的幂等键领取一条 AVAILABLE 数据，同一采集员可在同一或不同任务中同时持有多条普通待录制作业。三个页签固定为：`PENDING`（待录制和待返修）、`SUBMITTED`（可播放、重新录制并保存修改）、`FINISHED`（待审核和已完成，只读播放和查看文本）。每个页签在自身状态集合内统一按最新修改时间倒序、条目序号升序，不再按状态分组；服务端每页固定读取 10 条。提交修改若与审核领取并发并收到 `STALE_STATE`，页面刷新最新状态并提示状态已变化；待审核和已完成隐藏录制、文本编辑、提交及释放操作。

录音作业页使用固定高度的四状态卡片骨架：待录音、录音中、已暂停和处理中切换时，状态文字、波形及“暂停/继续/结束”控制不会上下跳动。麦克风、播放、暂停、继续、停止和垃圾桶均使用本地 Iconfont SVG；垃圾桶仍表示“释放任务”并保留确认弹窗。录制时按 PCM 帧 RMS 显示音量波形，并按累计 PCM 样本实时显示 `MM:SS`；暂停期间不累计，继续后接续，重录或错误归零，结束后由播放器保留最终时长。

实时波形只使用当前 PCM 帧，不保存历史音量或移动平均。RMS 不超过 `0.02` 时，7 根柱子全部保持 `8rpx` 平线；有声时按中心最高、左右对称的固定权重展开，RMS 达到 `0.25` 后按最高幅度显示，柱高限制在 `100rpx`。开始录音、暂停和错误时立即恢复平线，继续录音后由下一帧声音重新驱动。

参考音频、当前录制结果和已提交/只读结果统一使用本地 `audio-player` 组件，由 `wx.createInnerAudioContext()` 提供播放、暂停、拖动进度、结束复位、错误提示和页面卸载销毁。参考视频使用原生 `<video>`。URL-only 新条目直接播放 `referenceAudioUrl/referenceVideoUrl` 公网地址；历史条目没有 URL 字段时使用 `/api/media/public/reference/{mediaId}` 后端公网兼容地址，不再通过 `downloadFile` 临时目录加载参考媒体。生产 URL 必须来自微信后台配置的合法域名。参考音频、参考视频和已提交录音独立加载，单项失败不阻断作业主体或另一媒体。不同音频播放器也彼此独立，可同时播放；界面只显示一个动态时间值，不再使用原生 `<audio>` 或重复展示时长。WAV 采用流式写入，MP3 使用固定版本 `@breezystack/lamejs@1.2.7` 的本地 vendor 文件，运行时不依赖网络。

待录制和待返修作业在提交按钮上方显示胶囊样式“提交后自动领取下一条”开关，首次默认开启并用 `autoClaimNextEnabled` 本地记忆。开启时提交成功会使用稳定幂等键领取并 `redirectTo` 下一条；弱网重试复用同一键。池空、权限失效、任务停止或其他领取失败会 Toast 提示并进入当前任务数据页；关闭时返回上一页。已提交状态的“保存修改”不显示该开关，也不自动领取。

录音、参考音频、参考视频和录制结果完全独立，可同时运行并产生预期的声音混叠；开始或继续任一媒体不再暂停其他媒体。离开作业页后，音频组件、原生视频和录音会话仍分别按生命周期停止并释放资源。录音暂停继续保持平线且不写入 PCM。录音区外层最小高度为 `460rpx`、状态层为 `392rpx`，作业页专用文字输入框固定为 `200rpx`，不影响其他页面。登录、领取、提交、录音保存和播放等操作错误使用悬浮 Toast，不再在卡片底部追加错误文字；只有页面整体数据加载失败保留带“重新加载”的阻塞状态。作业页断网加载时显示“网络链接失败，请检查网络。”，其他业务加载错误仍保留服务端文案。

播放器初始化期间可能短暂收到空资源路径；空路径只复位组件显示，不得写入 `InnerAudioContext.src`，避免微信音频内核尝试解码空资源并产生 `Unable to decode audio data`。只有有效的本地临时文件路径才会交给音频上下文。

## 本地导入

1. 复制 `project.config.example.json` 为 `project.config.json`，在本地文件中填写测试 AppID。
2. 需要修改 API 地址时，新建不受 Git 跟踪的 `config/private.js`：

   ```js
   module.exports = { apiBaseUrl: 'https://your-test-api.example.com' }
   ```

3. 在微信开发者工具中导入 `apps/miniprogram`。本地 HTTP 联调可暂时关闭“不校验合法域名”；真机和正式环境必须使用 HTTPS，并在微信后台配置 request、uploadFile、downloadFile 以及参考音频/视频公网地址对应的合法域名。

AppSecret 只存在后端环境变量 `WECHAT_APP_SECRET`，绝不得放入小程序。`project.config.json`、`project.private.config.json` 和 `config/private.js` 均已忽略。

录音授权由实际调用 `wx.getRecorderManager()` 开始录音时触发；不要在 `app.json.permission` 中声明无效的 `scope.record`。麦克风被拒绝后的重新授权仍需在开发者工具和真机验证。

## 品牌头像

小程序名称使用“砚数声采”，头像资产位于：

- `assets/branding/yanshu-avatar-144.png`：144×144 PNG，文件小于 2MB，用于微信小程序后台上传。
- `assets/branding/yanshu-avatar.svg`：同构矢量源文件，用于代码引用或后续无损导出。

头像固定使用暖米白背景、墨黑录音声纹、朱砂红录音点和“砚数”文字。修改时应同步更新 PNG 与 SVG，并重新检查 48px 列表尺寸的可读性。

## 按钮圆角与下拉刷新

小程序全局清除微信原生 `button::after` 边框，普通文字按钮统一使用中等圆角矩形，迷你按钮使用独立尺寸；任务页筛选项保持胶囊形，播放、录音和录音控制按钮保持圆形。新增按钮时应复用现有基础类或明确覆盖尺寸与圆角，不要依赖原生按钮默认边框。

下拉刷新只在数据浏览页开启：任务大厅、任务数据和我的统计。任务数据页使用“上一页 / 可选择的第 X 页，共 Y 页 / 下一页”显式分页，可直接选择任意页；切换“待处理 / 已提交 / 已完成”页签时回到第一页，下拉刷新保持当前页，数据减少导致页码越界时自动回到新的最后一页，请求失败时保留当前列表。登录、资料设置和录音作业页不得开启下拉刷新，避免打断输入或录音。
