# AGENTS.md

本文件是当前仓库的项目协作规则。执行任何代码修改前必须优先阅读本文件、`README.md` 和 `docs/` 下与任务相关的文档。

## 当前阶段

- 当前阶段只做录音任务平台空项目框架和文档骨架。
- 不实现完整业务功能，不新增任务领取、录音上传、审核、登录、JWT 或权限配置逻辑。
- Spring Security 当前只作为依赖引入，后续登录/JWT 阶段再配置 `SecurityConfig`。

## 修改前要求

- 先检查 `git status`，确认已有改动，避免覆盖用户文件。
- 先阅读 `README.md`、`AGENTS.md` 和 `docs/` 下相关文档。
- 如果目标目录已有非占位内容，必须停止并报告，不允许覆盖。
- 不随意引入新依赖；如必须引入，先说明原因、替代方案、影响和验证方式。

## 文档同步

- 修改业务逻辑后必须同步更新相关文档。
- 修改接口时同步更新 `docs/04-api-design.md`。
- 修改数据库结构时同步更新 `docs/03-database-design.md`。
- 修改审核流程时同步更新 `docs/05-review-flow.md`。
- 每次较明确的 AI 辅助改动应更新 `docs/10-ai-change-log.md`。

## 安全要求

严禁提交以下内容：

- 密码、API Key、Token、Cookie、Authorization 头
- 真实用户隐私数据、员工敏感信息、客户数据、合同内容
- 未脱敏截图、完整签名 URL、完整音频 URL
- `.env` 中的真实配置

日志和文档只能保留必要摘要，例如状态码、请求 ID、hostname、错误摘要和耗时。

## 验证要求

每次修改后需要说明：

- 修改文件
- 验证命令和结果
- 风险点
- Git 状态

只运行项目真实存在的命令。验证失败时必须说明失败命令、原因、影响范围和下一步建议，不允许假装通过。

## Git 要求

- 默认不执行 `git add`、`git commit`、`git push`。
- 如用户明确要求提交，需在验证通过后再提交。
- 提交信息应简洁清晰，例如 `feat: initialize project scaffold`。
